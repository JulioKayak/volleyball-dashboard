3. Tipos principales

Días y géneros

export type Day = 'mon' | 'tue' | 'wed' | 'thu' | 'fri' | 'sat' | 'sun'
export type Gender = 'M' | 'F'

export const WORK_DAYS: Day[] = ['mon', 'tue', 'wed', 'thu', 'fri']

Disponibilidad semanal

interface TimeSlot          { start: string; end: string }   // "HH:MM"
interface DayAvailability   { available: boolean; slots: TimeSlot[] }
type WeeklyAvailability = Record<Day, DayAvailability>

Pabellón

interface Pavilion {
  id: string
  name: string
  maxCourts: number              // 1 | 2 | 3
  color: string                  // hex
  availability: WeeklyAvailability
}

Equipo

interface Team {
  id: string
  name: string
  category: string               // Escuelas, Benjamín, ... Senior
  gender: Gender
  sessionsPerWeek?: number       // nº de entrenos semanales (default 3)
}

Entrenador

interface Coach {
  id: string
  name: string
  teamIds: string[]              // equipos que entrena
  canDoubleCoach: boolean        // puede llevar 2 equipos a la vez
  availability: WeeklyAvailability
  playerTeamId?: string          // si también es jugador/a
}

Sesión de entrenamiento

interface Session {
  id: string
  pavilionId: string
  courtNumber: number
  teamId: string
  coachId: string
  day: Day
  startTime: string              // "HH:MM"
  endTime: string
}

5. Secciones de la app

🗓️ Horario semanal SchedulePage

Vista principal. Pestañas Lun–Vie, un panel por pabellón con columnas por pista y timeline vertical por horas.

  • Drag & drop de sesiones entre pistas y horas.
  • Sesiones restantes: sección colapsable con cards arrastrables (una por sesión pendiente según sessionsPerWeek); al soltar en una pista se crea la sesión (90 min por defecto).
  • Pabellones colapsables por día para ganar espacio visual.
  • Pabellones sin horario ese día quedan ocultos.
  • Sesiones en conflicto muestran un triángulo rojo; al pulsarlo se abre un popup con detalle del error.
  • Control de zoom (Compacto / Normal / Grande).

🏛️ Pabellones PavilionsPage

CRUD de pabellones. Cada uno con nombre, nº de pistas (1–3), color y disponibilidad semanal por franjas.

👥 Equipos TeamsPage

Agrupados por categoría (Escuelas → Senior). Filtro por género (M/F). Cada equipo: nombre, categoría, género, sesiones/semana y entrenadores asignados.

🎯 Entrenadores CoachesPage

Nombre, disponibilidad semanal, equipos que entrena, flag "doble" (dos equipos a la vez) y opcional "también jugador/a" con equipo de juego.

📤 Configuración / Export ExportPage

Exportar el estado a JSON e importar desde JSON (reemplaza todo).

6. Componentes clave

7. Motor de conflictos (utils/conflicts.ts)

Cada sesión se valida contra el resto. checkConflicts() devuelve un ConflictReport:

interface ConflictReport {
  coachDoubleBooked: boolean     // entrenador con demasiados equipos a la vez
  courtOccupied: boolean         // pista ocupada por otra sesión
  coachUnavailable: boolean      // fuera de la disponibilidad del entrenador
  pavilionUnavailable: boolean   // fuera del horario del pabellón
  coachPlayerConflict: boolean   // entrena mientras juega como jugador
}

El límite de "doble booking" del entrenador se relaja a 2 equipos si canDoubleCoach.

8. Flujo típico de uso

  1. Crear pabellones con su horario semanal y nº de pistas.
  2. Crear equipos indicando categoría, género y sesiones/semana.
  3. Crear entrenadores, asignarlos a equipos y configurar su disponibilidad.
  4. Ir a Horario. Expandir "Sesiones restantes" y arrastrar cards a las pistas correspondientes.
  5. Ajustar horas arrastrando sesiones dentro del grid.
  6. Revisar avisos rojos y editar sesiones en conflicto desde el popup.
  7. Exportar el JSON como backup desde Configuración.
